# [머니로그] 프로젝트 관리 계획서

## 1. 개요 (Introduction)
### 1.1 문서의 목적
*본 문서는 '머니로그' 프로젝트의 전반적인 관리 지표를 설정하여, 프로젝트가 차질 없이 목표를 달성할 수 있도록 가이드라인을 제공하는 것을 목적으로 합니다.

### 1.2 프로젝트 정의
* **머니로그(MoneyLog)**: 외부 은행 연동 없이 사용자가 직접 지출과 수입을 입력하여 개인정보를 안전하게 보호하는 심플한 자산 관리 모바일 앱입니다.

### 1.3 관련 문서 및 용어 정의
* **관련 문서**: 머니로그 요구사항정의서(project1.md)
* **용어 정의**: 로컬 DB(사용자 기기 내 저장소), API(기능 제어 인터페이스)

## 2. 개발 계획 (Development Plan)
### 2.1 프로세스 모델 및 개발 방법론
* **Agile (Scrum) 모델**: 빈번한 요구사항 변경에 빠르게 대응하고 팀원 간의 원활한 소통을 위해 스크럼 방식을 채택합니다.

### 2.2 작업 분할도 (WBS)
1. **요구분석**: 수입/지출 항목 정의, 카테고리 종류 확정 
2. **설계**: 통계 그래프 UI 디자인, 입력 화면 와이어프레임 작성 
3. **코딩**: 로컬 DB 연동 개발, 카테고리 커스터마이징 기능 구현, 통계 리포트 엔진 개발
4. **테스트 및 종료**: 데이터 계산 오류 확인, 최종 설치 파일(APK) 생성
